🔴 ثغرات حرجة (Critical)

1. كشف رسائل الأخطاء التفصيلية (Debug Mode) حرجة
config/app.php:42 | .env

الوصف:

وضع Debug مفعل في بيئة الإنتاج، مما يكشف رسائل الأخطاء التفصيلية stack traces للمستخدمين. هذا يسمح للمهاجمين برؤية هيكل التطبيق الداخلي.

المخاطر:

  • كشف مسارات الملفات الداخلية
  • كشف إصدارات المكتبات المستخدمة
  • كشف استعلامات قاعدة البيانات
  • تسهيل هجمات SQL Injection
  • تسهيل هجمات XSS

الحل:

في ملف .env:

APP_DEBUG=false
APP_ENV=production
2. عدم وجود Rate Limiting على تسجيل الدخول حرجة
routes/api.php | AuthController.php

الوصف:

لا توجد قيود على عدد محاولات تسجيل الدخول. هذا يسمح للمهاجمين بمحاولات لا نهائية لتخمين كلمات المرور (Brute Force Attack).

المخاطر:

  • هجمات Brute Force لتخمين كلمات المرور
  • هجمات Credential Stuffing
  • إ锁 حساب المستخدمين
  • استنزاف موارد الخادم

الحل:

أضف Rate Limiting في routes/api.php:

use Illuminate\Cache\RateLimiting\Limit;
use Illuminate\Support\Facades\RateLimiter;

RateLimiter::for('login', function (Request $request) {
    return Limit::perMinutes(5, 5)->by($request->ip());
});

Route::post('/auth/login', [AuthController::class, 'login'])
    ->middleware('throttle:login');

🟠 ثغرات عالية (High)

3. كشف معلومات المستخدم في UserResource عالية
app/Http/Resources/UserResource.php

الوصف:

Resource المستخدم يرجع بيانات حساسة مثل is_banned و ban_reason و banned_at لجميع المستخدمين. هذه المعلومات لا يجب أن تكون متاحة.

المخاطر:

  • كشف حالة الحظر للمستخدمين
  • كشف أسباب الحظر
  • كشف تواريخ الحظر
  • يمكن استغلالها لشن هجمات على المستخدمين المحظورين

الحل:

عدّل Resource للتحقق من صلاحية المستخدم:

public function toArray(Request $request): array
{
    $data = [
        'id' => $this->id,
        'name' => $this->name,
        'email' => $this->email,
        'phone' => $this->when(
            $request->user()?->isAdmin(), 
            $this->phone
        ),
        'role' => $this->role,
        'is_active' => $this->is_active,
        'email_verified_at' => $this->email_verified_at 
            ? $this->email_verified_at->toDateTimeString() 
            : null,
        'created_at' => $this->created_at->toDateTimeString(),
    ];

    // أضف حقول الحظر فقط للأدمن
    if ($request->user()?->isAdmin()) {
        $data['is_banned'] = $this->is_banned;
        $data['ban_reason'] = $this->when($this->is_banned, $this->ban_reason);
        $data['banned_at'] = $this->when($this->is_banned, 
            $this->banned_at?->toDateTimeString()
        );
    }

    return $data;
}
4. عدم التحقق من حالة حساب المستخدم في API عالية
bootstrap/app.php

الوصف:

المصادقة بـ Sanctum لا تتحقق تلقائياً من حالة is_active أو is_banned. المستخدم المحظور أو المعطل يمكنه استخدام Token القديم.

المخاطر:

  • المستخدم المحظور يمكنه استخدام النظام
  • المستخدم المعطل يمكنه استخدام النظام
  • لا يوجد فصل تلقائي للجلسة

الحل:

أضف Middleware للتحقق من حالة الحساب:

// app/Http/Middleware/CheckUserStatus.php
public function handle(Request $request, Closure $next)
{
    $user = $request->user();
    
    if (!$user || !$user->is_active) {
        $request->user()->currentAccessToken()->delete();
        return response()->json([
            'success' => false,
            'message' => 'الحساب غير نشط'
        ], 403);
    }
    
    if ($user->is_banned) {
        $request->user()->currentAccessToken()->delete();
        return response()->json([
            'success' => false,
            'message' => 'الحساب محظور'
        ], 403);
    }
    
    return $next($request);
}
5. عدم وجود Rate Limiting على API العام عالية
routes/api.php

الوصف:

جميع واجهات API العامة (العقارات، المناطق، البحث) لا تحتوي على Rate Limiting. هذا يسمح بهجمات DoS و scraping.

المخاطر:

  • هجمات DoS/DDoS
  • استخراج جميع البيانات (Data Scraping)
  • استنزاف موارد الخادم
  • تكلفة عالية على الاستضافة السحابية

الحل:

أضف Rate Limiting عام:

RateLimiter::for('api', function (Request $request) {
    return Limit::perMinute(60)->by($request->ip());
});

// في routes/api.php
Route::middleware('throttle:api')->group(function () {
    // الواجهات العامة
});
6. عدم تنظيف Base64 Images عالية
app/Services/PropertyService.php:142-166

الوصف:

دالة addImagesFromBase64 تقبل Base64 بدون تحقق كافٍ. يمكن رفع ملفات ضارة.

المخاطر:

  • رفع ملفات ضارة (Web Shell)
  • رفع ملفات كبيرة جداً
  • التحايل على قيود نوع الملف

الحل:

public function addImagesFromBase64(Property $property, array $base64Images): void
{
    $existingCount = $property->images()->count();
    $maxSize = 2 * 1024 * 1024; // 2MB
    $allowedTypes = ['image/jpeg', 'image/png', 'image/gif', 'image/webp'];
    
    foreach ($base64Images as $base64) {
        if (!$base64) continue;
        
        // استخراج نوع الملف
        if (preg_match('/^data:image\/(\w+);base64,/', $base64, $typeMatch)) {
            $imageType = 'image/' . $typeMatch[1];
            $base64Data = preg_replace('/^data:image\/\w+;base64,/', '', $base64);
        } else {
            continue; // صيغة غير صالحة
        }
        
        // التحقق من نوع الملف
        if (!in_array($imageType, $allowedTypes)) {
            continue;
        }
        
        $imageData = base64_decode($base64Data);
        
        // التحقق من حجم الملف
        if (strlen($imageData) > $maxSize) {
            continue;
        }
        
        // التحقق من MIME type الفعلي
        $finfo = new \finfo(FILEINFO_MIME_TYPE);
        $actualMime = $finfo->buffer($imageData);
        
        if (!in_array($actualMime, $allowedTypes)) {
            continue;
        }
        
        $filename = time() . '_' . uniqid() . '.jpg';
        $path = 'properties/' . $property->id . '/' . $filename;
        
        \Illuminate\Support\Facades\Storage::disk('public')->put($path, $imageData);
        
        PropertyImage::create([
            'property_id' => $property->id,
            'image_path' => $path,
            'is_primary' => $existingCount === 0,
        ]);
        
        $existingCount++;
    }
}
7. عدم وجود تحكم في الوصول CORS عالية
bootstrap/app.php

الوصف:

لا يوجد ملف cors.php أو إعدادات CORS. هذا يسمح لأي موقع باستدعاء API.

المخاطر:

  • هجمات CSRF
  • استخدام API من مواقع ضارة
  • سرقة بيانات المستخدمين
  • الربط بنطاقات غير مصرح بها

الحل:

أنشئ config/cors.php:

return [
    'paths' => ['api/*'],
    'allowed_methods' => ['*'],
    'allowed_origins' => [
        'https://yourdomain.com',
        'https://www.yourdomain.com',
    ],
    'allowed_origins_patterns' => [],
    'allowed_headers' => ['*'],
    'exposed_headers' => [],
    'max_age' => 0,
    'supports_credentials' => false,
];

🔵 ثغرات متوسطة (Medium)

8. عدم التحقق من كلمة المرور القوية متوسطة
app/Http/Requests/AuthRequest.php:22

الوصف:

التحقق من كلمة المرور يسمح بـ 8 أحرف فقط دون تحقق من التعقيد. كلمات مرور ضعيفة مثل "12345678" مقبولة.

الحل:

'password' => [
    'required',
    'string',
    'min:8',
    'confirmed',
    'regex:/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&])[A-Za-z\d@$!%*?&]/',
],

يجب أن تحتوي على: حرف كبير، حرف صغير، رقم، و رمز خاص.

14. عدم وجود رؤوس أمنية HTTP منخفضة
bootstrap/app.php

الوصف:

لا توجد Middleware لإضافة رؤوس أمنية مثل CSP, X-Frame-Options, X-Content-Type-Options.

الحل:

أنشئ Middleware:

// app/Http/Middleware/SecurityHeaders.php
public function handle(Request $request, Closure $next)
{
    $response = $next($request);
    
    $response->headers->set('X-Frame-Options', 'SAMEORIGIN');
    $response->headers->set('X-Content-Type-Options', 'nosniff');
    $response->headers->set('X-XSS-Protection', '1; mode=block');
    $response->headers->set('Referrer-Policy', 'strict-origin-when-cross-origin');
    
    return $response;
}

💡 التوصيات العامة

1. مصادقة ثنائية (2FA)

أضف تحققاً ثنائياً للحسابات الحساسة (الأدمن).

2. تشفير البيانات الحساسة

شفر أرقام الهواتف وغيرها من البيانات الشخصية.

3. سجلات التدقيق (Audit Logs)

سجل جميع العمليات الحساسة (تغيير الأدوار، الحظر، حذف البيانات).

4. النسخ الاحتياطي التلقائي

أعد جدولة نسخ احتياطية منتظمة لقاعدة البيانات.

5. فحص أمني دوري

أجرِ فحصاً أمنياً منتظماً باستخدام أدوات مثل OWASP ZAP.

6. HTTPS فقط

أجبر استخدام HTTPS مع HSTS.

7. حماية من Brute Force

أضف حماية إضافية لتسجيل الدخول مع IP blocking.

8. التحقق من البريد الإلكتروني

أضف تحققاً إلزامياً من البريد الإلكتروني قبل السماح باستخدام الحساب.
