1 المصادقة (Authentication)

POST /auth/register تسجيل مستخدم جديد
ℹ️
إنشاء حساب مستخدم جديد في النظام. الدور الافتراضي هو user ما لم يُحدد غير ذلك.

Headers

المفتاح القيمة الوصف
Accept application/json نوع المحتوى المطلوب
Content-Type application/json نوع البيانات المرسلة

Body (Request Body)

المعامل النوع مطلوب الوصف
name string نعم اسم المستخدم الكامل
email email نعم البريد الإلكتروني (يجب أن يكون فريداً)
phone string نعم رقم الهاتف السوري (يجب أن يبدأ بـ +963 أو 09)
password string نعم كلمة المرور (8+ أحرف، يجب أن تحتوي أحرف وأرقام)
password_confirmation string نعم تأكيد كلمة المرور
role string لا الدور: user, owner (الافتراضي: user)
مثال على الطلب JSON 
{
  "name": "أحمد محمد",
  "email": "ahmed@example.com",
  "phone": "+963944123456",
  "password": "Password123",
  "password_confirmation": "Password123",
  "role": "owner"
}
200 - نجاح 422 - خطأ في التحقق
200 OK - الاستجابة الناجحة JSON 
{
  "success": true,
  "message": "تمت العملية بنجاح",
  "token": "1|abc123xyz...",
  "data": {
    "id": 1,
    "name": "أحمد محمد",
    "email": "ahmed@example.com",
    "phone": "+963944123456",

  }
}
422 Validation Error - خطأ في التحقق JSON

  "success": false,
  "message": "بيانات غير صالحة",
  "errors": {
    "email": [
      "البريد الإلكتروني مستخدم مسبقاً"
    ],
    "phone": [
      "رقم الهاتف يجب أن يكون بصيغة سورية صحيحة"
    ],
    "password": [
      "كلمة المرور يجب أن تحتوي 8 أحرف على الأقل مع أحرف وأرقام"
    ]
  }
}

  "message": "بيانات الاعتماد غير صحيحة"

  "message": "الحساب محظور. يرجى التواصل مع الإدارة"

3 العقارات (Properties)

GET /properties عام

Query Parameters

المعامل النوع الوصف
per_page integer عدد النتائج لكل صفحة (الافتراضي: 15)
type string النوع: sale, rent
property_type string نوع العقار: apartment, villa, house, land, shop, office, warehouse, building
region_id integer معرف المنطقة
min_price integer الحد الأدنى للسعر
max_price integer الحد الأقصى للسعر
min_area integer الحد الأدنى للمساحة
max_area integer الحد الأقصى للمساحة
status string الحالة: active, pending, sold, rented
sort_by string الترتيب: created_at, price, area
sort_order string نوع الترتيب: asc, desc

POST /properties Owner/Admin
⚠️
القيود:
  • فقط الأدوار owner أو admin يمكنها إنشاء عقارات
  • الحد الأقصى 400 عقار للمكاتب العقارية (owner)
  • العقار يُنشأ بحالة pending ويحتاج موافقة الإدارة
المعامل النوع مطلوب الوصف
title string نعم عنوان العقار (255 حرف كحد أقصى)
description string نعم وصف العقار (5000 حرف كحد أقصى)
price numeric نعم السعر (يجب أن يكون أكبر من صفر)
type string نعم النوع: sale أو rent
property_type string نعم نوع العقار
area numeric نعم المساحة (بالمتر المربع، يجب أن تكون أكبر من صفر)
region_id integer نعم معرف المنطقة
location string نعم الموقع النصي
latitude float لا خط العرض (-90 إلى 90)
longitude float لا خط الطول (-180 إلى 180)
images[] file لا صور العقار (10 صور كحد أقصى، كل صورة 2MB)
مثال على الطلب JSON 
{
  "title": "شقة فاخرة 3 غرف",
  "description": "شقة جديدة طابق رابع، جديدة لم تسكن، قرب الخدمات",
  "price": 50000000,
  "type": "sale",
  "property_type": "apartment",
  "area": 150,
  "region_id": 1,
  "location": "المزة - دمشق",
  "latitude": 33.5138,
  "longitude": 36.2765
}

4 طلبات التواصل (Contact Requests)

POST /contact مطلوب
⚠️
القيود:
  • لا يمكن لمالك العقار إرسال طلب تواصل لعقاره
  • لا يمكن إرسال طلب متكرر لنفس العقار
  • الرسالة محدودة بـ 1000 حرف
المعامل النوع مطلوب الوصف
property_id integer نعم معرف العقار
name string نعم اسم مقدم الطلب
phone string نعم رقم الهاتف
message string لا الرسالة (1000 حرف كحد أقصى)
